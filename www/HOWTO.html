2. Installation

Method A: Direct Open (Easiest)

  1. Unzip the morse-kids-lab-v1.2.zip archive
  2. Open index.html by double-clicking it in your file manager
  3. That's it! The app runs entirely in your browser

⚠️ BLE will NOT work when opening via file:// in most browsers. Use Method B for BLE.

Method B: Local Server (Required for BLE)

Web Bluetooth needs a secure context (HTTPS or localhost).

Option 1: Python (pre-installed on Mac/Linux)

# Navigate to the app folder
cd morse-kids-lab-v1.2

# Start server
python3 -m http.server 8000

# Open in browser
# → http://localhost:8000

Option 2: Node.js

# Install serve globally (one time)
npm install -g serve

# Navigate and run
cd morse-kids-lab-v1.2
serve .

# Open in browser
# → http://localhost:3000

Option 3: PHP

cd morse-kids-lab-v1.2
php -S localhost:8000

# → http://localhost:8000

Option 4: VS Code Live Server Extension

  1. Install the Live Server extension in VS Code
  2. Open the morse-kids-lab-v1.2 folder in VS Code
  3. Right-click index.html → Open with Live Server
  4. Browser opens automatically at http://127.0.0.1:5500

Method C: Host Online

Upload all files to any web hosting (GitHub Pages, Netlify, Vercel, etc.):

# GitHub Pages example
git init
git add .
git commit -m "Morse Kids Lab v1.2"
git remote add origin https://github.com/YOUR_USERNAME/morse-kids-lab.git
git push -u origin main
# Enable Pages in repository settings → Source: main branch

4. App Tour — All 8 Tabs

📚 Tab 1: Learn

What: Interactive Morse code reference chart.

How to use: 1. Browse all 36 characters (A–Z, 0–9) 2. Click any character to hear its Morse code as audio beeps 3. Read the level explanation (Newb/Explorer/Developer) for context 4. Characters show the letter and its dot/dash pattern

Try this: Click S (···) then O (———) then S (···) — you just heard SOS!

✏️ Tab 2: Encode

What: Type text → see and hear it as Morse code.

How to use: 1. Type text in the input field (e.g., "HELLO") 2. Watch the Morse output update in real-time below 3. Click ▶ Play to hear it as audio beeps 4. Watch the lamp flasher blink with each dot and dash 5. Adjust Speed (WPM) with the slider 6. Click 📋 Copy to copy the Morse to clipboard 7. Click 📡 Send to micro:bit if BLE is connected

🔍 Tab 3: Decode

What: Paste Morse code → read the decoded text.

How to use: 1. Type or paste Morse code using: - . or · for dot (dit) - - or — for dash (dah) - Space between letters - / between words 2. Example: .... . .-.. .-.. --- / .-- --- .-. .-.. -.. 3. Decoded text appears below in real-time 4. Click 🔊 Play to hear the Morse code

👆 Tab 4: Tap Lab

What: Become a telegraph operator! Tap Morse code manually.

How to use: 1. Press the big TAP button 2. Short tap (< 200ms) = dot (dit ·) 3. Long press (≥ 200ms) = dash (dah —) 4. Wait 600ms between letters (auto-detected) 5. Wait 1400ms between words (auto-detected) 6. Watch the waveform build up as you tap 7. See your decoded text appear below

Challenge: Try tapping SOS — three short, three long, three short!

🔊 Tab 5: Listen Station

What: Type a message and hear it as Morse audio with controls.

How to use: 1. Type your message in the input field 2. Adjust Frequency (pitch: 400–1000 Hz, default 700 Hz) 3. Adjust Speed (5–30 WPM, default 15 WPM) 4. Click ▶ Play to hear the message 5. Watch the visual timeline — dots and dashes light up as they play 6. Click ⏹ Stop to cancel playback

Pro tip: 700 Hz is the classic CW (Continuous Wave) frequency used by amateur radio operators!

🤖 Tab 6: micro:bit Lab

What: Connect a real BBC micro:bit via Bluetooth and exchange Morse code.

Three sections:

A. BLE Connection 1. Click 📡 Connect to scan for your micro:bit 2. Select your micro:bit from the browser popup 3. Status changes to green "Connected" 4. See device name and connection info

B. Simulator - Use the on-screen SVG micro:bit even without hardware - Click A button = dot · (LED flashes small diamond) - Click B button = dash — (LED flashes big diamond) - Click 🔄 Reset to clear

C. Send/Receive - Type a message → click 📡 Send to send Morse to micro:bit - micro:bit buzzes and flashes each dot/dash - Press buttons A/B on physical micro:bit → app receives and decodes

D. Firmware Code - Expand the MicroPython section to see the firmware - Click 📋 Copy Code to copy it - Flash to your micro:bit (see Section 5 below)

🏆 Tab 7: Challenge Zone

What: Test your Morse knowledge with 4 game modes!

Game modes:

Mode How it works
🔤 Letter Quiz See a Morse pattern → pick the correct letter from 4 choices
📝 Word Decode Decode a complete word from Morse → type your answer
⚡ Speed Round Same as Letter Quiz but with a 10-second timer!
🆘 SOS Challenge Type the SOS Morse pattern correctly (··· ——— ···)

Scoring: - Correct answer = +10 points (+20 in Speed Round) - Streak counter tracks consecutive correct answers - Best score saved locally (survives browser restart) - 3+ streak = 🔥 Awesome streak! message

❓ Tab 8: Help

What: Reference material and guides.

5. micro:bit BLE Setup

Step-by-Step micro:bit Firmware Installation

What You Need

  • BBC micro:bit (V1.5 or V2)
  • USB cable (micro-USB for V1, USB-C for V2)
  • Computer with Chrome or Edge browser

Step 1: Get the Firmware Code

  1. Open Morse Kids Lab in your browser
  2. Go to the 🤖 micro:bit tab
  3. Scroll down to "micro:bit Firmware (MicroPython)"
  4. Click the expand arrow to reveal the code
  5. Click 📋 Copy Code

Step 2: Flash the Firmware

Option A: python.microbit.org (Recommended)

  1. Go to python.microbit.org
  2. Delete the default code in the editor
  3. Paste the copied firmware code
  4. Connect your micro:bit via USB
  5. Click Send to micro:bit
  6. Wait for the yellow LED to stop flashing

Option B: Mu Editor

  1. Download Mu Editor
  2. Select "BBC micro:bit" mode
  3. Paste the firmware code
  4. Click Flash

Step 3: Enable Bluetooth

After flashing: 1. Disconnect USB (the micro:bit runs on battery or USB power) 2. The micro:bit is now advertising its Bluetooth name 3. You should see "BBC micro:bit [xxxxx]" when scanning

Step 4: Connect from the App

  1. Open Morse Kids Lab at http://localhost:8000 (or HTTPS URL)
  2. Go to 🤖 micro:bit tab
  3. Click 📡 Connect
  4. A browser popup shows available devices
  5. Select your micro:bit (e.g., "BBC micro:bit [zutev]")
  6. Click Pair if prompted
  7. Status pill turns green: "Connected"

Step 5: Test It!

  • Type "SOS" in the Send field → click 📡 Send
  • Your micro:bit buzzes ··· ——— ··· and flashes the LEDs!
  • Press Button A on micro:bit → app shows a dot
  • Press Button B on micro:bit → app shows a dash

Wiring (Optional External Buzzer)

For micro:bit V1 (no built-in speaker):

micro:bit Pin 0 ──── Buzzer + (positive)
micro:bit GND   ──── Buzzer − (negative)

micro:bit V2 has a built-in speaker — no wiring needed!

10. Troubleshooting

BLE Won't Connect

Problem Solution
No "Connect" popup Use Chrome or Edge (Firefox/Safari don't support Web Bluetooth)
micro:bit not listed Make sure micro:bit is powered on and not connected to another device
"UART characteristics not found" Re-flash the MicroPython firmware — the UART service may not be enabled
Connection drops Stay within ~10m range; avoid other BLE interference
"Web Bluetooth not available" Make sure you're on localhost or HTTPS (not file://)

Audio Not Playing

Problem Solution
No sound on first click Click anywhere on the page first — browsers require a user gesture for audio
Very quiet Check device volume; try adjusting the Frequency slider in Listen tab
Distorted Lower the frequency or speed

App Issues

Problem Solution
Layout broken Try a different browser or clear cache
RTL not working Make sure you selected عربي from the Language dropdown
Scores lost Scores use localStorage — clearing browser data resets them
Fonts not loading You need internet on first load for Google Fonts

11. For Parents & Teachers

Educational Value

Morse Kids Lab teaches:

  • Encoding fundamentals — how information is represented in different forms
  • Binary concepts — Morse uses only 2 symbols, like computers use 0 and 1
  • Timing & rhythm — precision in communication
  • IoT basics — real wireless communication with micro:bit
  • Security thinking — encoding vs. encryption, physical security, consent
  • Problem solving — decoding challenges and pattern recognition

Classroom Use

  • Works on school Chromebooks (Chrome supports Web Bluetooth)
  • No installation needed — just serve from teacher's computer or school network
  • No internet required after first load
  • No accounts, no data collection, no tracking
  • Students can work individually or in pairs (one sends, one receives via BLE)
  • Challenge Zone provides built-in assessment
  • 3 skill levels allow differentiation within same activity

Safety

  • No network access — everything is local
  • No user data stored — only challenge scores in localStorage
  • BLE is consent-based — browser always asks permission
  • Content is age-appropriate — clean humor, respectful tone
  • Islam-friendly — respectful of cultural values

12. FAQ

Q: Do I need a micro:bit to use the app? A: No! The app works fully without hardware. The simulator provides a virtual micro:bit experience. BLE connection is optional.

Q: Does it work offline? A: Yes, after the first load (which fetches Google Fonts). All logic is local.

Q: Can two micro:bits communicate? A: The current firmware supports app ↔ micro:bit communication. For micro:bit ↔ micro:bit, you'd need the radio module (future feature).

Q: Is my data safe? A: Absolutely. No data leaves your device. No servers, no analytics, no cookies.

Q: Can I modify the app? A: Yes! It's pure HTML/CSS/JS — edit with any text editor.

Q: What's the recommended age? A: Ages 8–15, but the 3 skill levels make it accessible to younger kids and interesting for older ones.

Q: Why does BLE need Chrome? A: Web Bluetooth API is currently only supported in Chromium-based browsers (Chrome, Edge, Opera).
